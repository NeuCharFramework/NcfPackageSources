# Domain folder
  
`Domain`The folder contains the core business logic and domain model of the system. These models reflect the core concepts and rules of the business and provide the foundation for the entire system.`Domain`Folders typically contain the following types of components:
  
- Entity: a business object with a unique identifier whose properties and behavior change over time.
- Value Object: A business object without a unique identifier, used to represent the characteristics of a concept or entity, and its properties are immutable.
- Aggregation: A group of related objects composed of entities and value objects that are persisted and processed as a whole.
- Aggregate Root: The entry point of the aggregate, responsible for maintaining the integrity and consistency of the aggregate.
- Domain Service: Encapsulates operations related to business logic and usually does not belong to a single entity or value object.
- Domain Event: represents important business events that occur in the domain and is used to achieve decoupling and tracking business changes.
  
During development, be sure to place your core business logic and domain model in`Domain`folder to keep the code clear and maintainable.

## Domain folder structure
  
exist`Domain`In the folder, we have created 3 subfolders by default:`Migrations`、`Models`and`Services`, to organize different types of domain components.
  
### Migrations folder
  
`Migrations`The folder is used to store database migration files related to the domain model. These files are responsible for applying domain model changes to the database to keep the database structure and domain model in sync.

Normally, this folder is automatically generated by the XncfBuilder module, and manual modification is not recommended.
  
### Models folder
  
`Models`Folders are used to store domain model components such as entities, value objects, aggregates, and aggregate roots. These components reflect the core concepts and rules of the business and form the basis of the system.
  
### Services folder
  
`Services`Folders are used to store domain services, domain events and other operations related to business logic. These operations typically do not belong to a single entity or value object, but involve the interaction of multiple domain model components.
  
Please make sure that during development, you place different types of domain components in corresponding subfolders to keep your code clear and maintainable.
