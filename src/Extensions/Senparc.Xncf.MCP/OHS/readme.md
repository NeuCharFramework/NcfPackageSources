# Open Host Service (OHS)

In the NCF system, we use the Open Host Service (OHS) pattern from Domain-Driven Design (DDD) for communication between internal modules or domains and with external systems. OHS provides decoupled, integration-friendly communication through public, uniform interfaces (such as APIs and events). OHS has two branches: [Local](Local/readme.md) and [Remote](Remote/readme.md).
